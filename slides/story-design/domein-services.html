Vaak voorkomende patronen voor domein services

Facade:

    Kenmerken:
  • Vermindert complexiteit
  • Bundelt een proces
Een facade wordt gebruikt om een complex subsysteem een gemakkelijk te gebruiken interface te geven. Hierdoor moet de developer niet telkens opnieuw weten wat elke substap is van dit systeem maar kan dit door vragen aan de facede

Resource/controller:

    Kenmerken:
  • Binnenkomende service
  • Beschermd uw domein
Een Resource of controller wordt gebruikt om te bepalen wat er binnen buiten gaat vanuit het systeem

Repository:

    Kenmerken:
  • Abstractie voor het persisteren van data
Een Repository is een abstractie voor het persisteren van Data

Gateway:

    Kenmerken:
  • Abstractie voor externe systemen
Een Repository is een abstractie voor het persisteren van Data
